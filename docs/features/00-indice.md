# ⚠️ DEPRECADO — el tracking de features vive en GitHub Issues

> El **seguimiento, el estado, la prioridad y el backlog** de features se gestionan **en GitHub Issues** (épicas + sub-issues, labels, plantillas y el protocolo de _claim_ — ver `AGENTS.md`). Estas fichas quedan como **referencia histórica congelada**: no se añaden ni se editan. Para trabajar una feature, abre o coge su **issue**.
>
> Backlog vivo: **[GitHub Issues](https://github.com/GlobalEmergency/ResponseGrid/issues)**.
